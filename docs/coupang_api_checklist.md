# 쿠팡 API 적용 우선순위 체크리스트

마지막 업데이트: 2026-04-07  
작성자: Modiba

## P0 - 즉시 점검(운영 차단 이슈)

- [ ] `COUPANG_ACCESS_KEY`, `COUPANG_SECRET_KEY`가 Wing에서 발급된 최신 한 쌍인지 확인
- [ ] 해당 키의 Open API 권한이 실제로 활성화되어 있는지 확인
- [ ] Railway production Variables에 쿠팡 키가 정확히 반영되었는지 확인
- [ ] 401 오류 발생 시 원인을 키/권한 문제와 코드/서명 문제로 분리해 확인
- [ ] 테스트 시 연속 호출을 피하고 호출 간격을 두어 제한 정책 준수

## P1 - 이번 주 반영 권장(안정성 강화)

- [ ] 앱 사이드바에 쿠팡 키 사전 검증 상태(성공/실패) 표시
- [ ] `401`, `403`, `429`, `5xx` 응답코드별 사용자 안내 문구 분리
- [ ] 일시적 네트워크 오류(타임아웃/연결 실패)에 대해 짧은 백오프 재시도 적용
- [ ] 운영 환경에서 쿠팡 호출 On/Off 제어 변수(`COUPANG_ENABLED`) 도입
- [ ] 로그에 키/서명 원문이 절대 출력되지 않도록 보안 점검

## P2 - 다음 단계(품질/운영성)

- [ ] CEA 서명 문자열, 엔드포인트 경로, 쿼리 구성에 대한 단위 테스트 추가
- [ ] 응답코드별 실패율 및 호출 성공률 관측 지표 정리
- [ ] 샌드박스 부재(실환경만 존재) 전제와 점검 절차를 운영 문서에 명시
- [ ] 방화벽/네트워크 정책(출구 IP 포함) 점검 절차 문서화

## 실행 순서

1. P0 항목을 모두 확인하여 운영 차단 요소를 먼저 제거한다.
2. 쿠팡 키 검증이 안정화되면 P1 항목을 반영해 운영 안정성을 높인다.
3. 이후 P2 항목(테스트/관측/문서화)을 순차 적용해 장기 운영 품질을 고도화한다.
